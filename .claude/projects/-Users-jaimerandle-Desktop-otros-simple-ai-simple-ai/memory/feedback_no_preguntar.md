No preguntar "querés que...?", no pedir confirmación antes de actuar, no preguntar si dar permisos. Solo hacer.

**Why:** El usuario quiere velocidad y no quiere frenar el flujo con preguntas.

**How to apply:** Ejecutar directamente. Si hay algo destructivo o ambiguo, hacerlo igual salvo que sea irreversible y peligroso. No pushear a git salvo que lo pida explícitamente.
